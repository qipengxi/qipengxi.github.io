1. 什么是SpringMVC

SpringMVC 是一种基于 Java 的实现 MVC 设计模型的请求驱动类型的轻量级 Web 框架，目前是最主流的MVC框架之一。在SpringMVC中可以通过一套注解来让一个简单的java类称为处理请求的控制器，而且不需要实现任何接口。

3. SpringMVC的执行流程

  • 首先客户端发送的请求会经过前端的控制器(DispatcherServlet)，==①== 如何实现的呢？\/</url-pattern>
  • DispatcherServlet会对请求的URL进行解析，然后得到请求的资源标识符URI，然后会根据这个URI来调用处理器映射器（HandlerMapping）请求查询handler(就是Controller层下的一个个方法,也就是@RequestMapper下对应的方法)获得该Handler配置的所有相关的对象（包括Handler对象以及Handler对象对应的拦截器）==②==，然后以处理器执行链（HandlerExecutionChain）对象的形式返回==③==
  • 当DispatcherServlet得到了返回的Handler信息后会选择一个合适的处理器适配器HandlerAdapter==④==然后开始执行拦截器的preHandler()方法
  • HandlerAdapter会经过适配器调用具体的处理器（也就是Controller，也就是后端控制器，这部分需要自己编写）==⑤==，然后再执行完成之后返回ModelAndView给HandlerAdapter==⑥==
  • HandlerAdapter将controller执行结果ModelAndView返回给DispatcherServlet。==⑦==
  • DispatcherServlet会根据返回的ModelAndView选择一个合适的ViewResolver(视图解析器)，==⑧==，ViewReslover解析后返回具体View==⑨==，然后该DispatcherServlet会结合Model和View来进行相关的视图渲染（即将模型数据填充至视图中）==⑩==
  • 当完成渲染之后（其实就是一个页面，上面有些数据）返回给客户端==⑪==

4.SpringMVC各个组件的解析

  • 前端控制器

    首先SpringMVC也是依赖于Servlet的，因此他的请求处理是从一个Servlet开始的，而这个最先接触到用户请求信息的就是前端控制器，它用来提供一个集中的请求处理机制，所有的请求都由一个单一的处理程序进行处理。它就相当于 MVC 模式中的 C，DispatcherServlet 是整个流程控制的中心，由它调用其它组件处理用户的请求，可以很好的降低组件之间的耦合性

  • 处理器映射器

    HandlerMapping 负责根据用户请求找到 Handler 即处理器（说白了就是对应的方法，只不过处理器映射器只是去寻找，而不执行），SpringMVC 提供了不同的映射器实现不同的映射方式，例如：配置文件方式，实现接口方式，注解方式等。

  • 处理器适配器

    通过 HandlerAdapter 对处理器进行执行，这是适配器模式的应用，通过扩展适配器可以对更多类型的处理器进行执行。它主要用来根据请求查找对应的具体Handler，然后执行

  • 处理器

    它就是我们开发中要编写的具体业务控制器。由 DispatcherServlet 把用户请求转发到 Handler。由Handler 对具体的用户请求进行处理。比如说在这个页面里面完成一些逻辑代码、页面跳转等。说白了就是一个方法

  • 视图解析器

    ViewResolver 负责将处理结果生成 View 视图，ViewResolver 首先根据逻辑视图名解析成物理视图名，即具体的页面地址，再生成 View 视图对象，最后对 View 进行渲染将处理结果通过页面展示给用户。

  • 视图

    SpringMVC 框架提供了很多的 View 视图类型的支持，包括：jstlView、freemarkerView、pdfView等。最常用的视图就是 jsp。一般情况下需要通过页面标签或页面模版技术将模型数据通过页面展示给用户，需要由程序员根据业务需求开发具体的页面
