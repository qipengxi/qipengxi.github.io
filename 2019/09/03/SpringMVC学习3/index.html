2. SpringMVC的响应

SpringMVC的响应有如下类型：

  • 直接进行页面跳转
    • 直接返回字符串
    • 返回ModelAndView
  • 回写数据，不跳转页面
    • 直接返回字符串
    • 返回对象或者集合

2.1 页面跳转–返回字符串

  • ==没有在spring-mvc.xml里配置视图解析器的时候,应当直接返回页面所在目录：return “/jsp/success.jsp”;==
  • ==配制了视图解析器的时候：==
    • return “forward:页面名称”;//默认的是请求转发，路径不会变
    • return “redirect:页面名称”;//可以配置成重定向，路径会变
    • return “页面名称”代表默认，也就是省略了forward

2.4 写回数据–返回对象或者集合：SpringMVC自动返回对象的JSON格式字符串

但是在这之前，需要先告诉框架我们想要它帮我们转换JSON格式，在配置转换器的时候指定一个JSON转换器

==不过这样的配置太麻烦了，可以直接使用mvc的注解驱动代替上述配置：mvc:annotation-driven/==

@RequestParam/@PathVariable

  • RequestParam

    • 当请求的参数名称和Controller的业务方法参数名称不一致的时候，进行一个映射

  • @PathVariable(name=””)：Restful风格

    • 能够接收请求路径中占位符的值，并进行相应的映射。（路径中的占位符会被传递到value的属性值上，然后映射给username这个形参，最后可以通过打印语句输出）
