1. MyBatis核心配置文件中的层级关系

在MyBatis的配置文件中根标签是，在该根标签下又有很多的字标签，其中这些字标签的出现的数序并不是任意的。

  • configuration（必须有啊）
  • properties
  • settings
  • typeAliases
  • typeHandlers
  • objectFactory
  • plugins
  • environments
    • environment
      • transactioManager
      • dataSource
  • databaseIdProvider
  • mappers
    • mapper
    • package

2. 常用配置浅析

2.1 environments标签

这个标签是用来进行数据库环境的配置，它支持多环境的配置。

  • environments：告诉容器我这个标签是用来配置数据库环境的
  • environment：具体的数据库环境
  • transactionManager：事务管理器，用来指定事务的管理类型
    • JDBC：这个配置就是直接使用了JDBC 的提交和回滚设置，它依赖于从数据源得到的连接来管理事务作用域。
    • MANAGED：这个配置几乎没做什么。它从来不提交或回滚一个连接，而是让容器来管理事务的整个生命周期（比如 JEE 应用服务器的上下文）。 默认情况下它会关闭连接，然而一些容器并不希望这样，因此需要将 closeConnection 属性设置为 false 来阻止它默认的关闭行为。
  • dataSource：指定当前数据源类型
    • POOLED：这种数据源的实现利用“池”的概念将 JDBC 连接对象组织起来。
    • UNPOOLED：这个数据源的实现只是每次被请求时打开和关闭连接。
    • JNDI：这个数据源的实现是为了能在如 EJB 或应用服务器这类容器中使用，容器可以集中或在外部配置数据源，然后放置一个 JNDI 上下文的引用。

2.2 mappers标签（该标签下有mapper以及package两个字标签供选择）

  • 该标签的作用是在核心配置文件中加载映射文件的的，加载方式有如下几种：

    • 使用相对于类路径的资源引用（xml方式，这个时候具体的sql语句是写在映射文件里面的）

    • 使用完全限定资源定位符（URL）（xml方式，这个时候具体的sql语句是写在映射文件里面的）

    • 使用具体接口实现类的完全限定类名，这样一来具体的某个具体的接口就会被注册成映射器（注解方式，这个时候具体的sql语句是使用注解写在接口的方法上的）

    • 将包内的映射器接口实现全部注册为映射器（注解方式，这个时候具体的sql语句是使用注解写在接口的方法上的）

2.3 Properties标签

实际开发中，习惯将数据源的配置信息单独抽取成一个properties文件，该标签可以加载额外配置的properties文件

2.4 typeAliases标签

这个标签主要用来声明别名。

2.5 typeHandlers标签

这个标签无论是 MyBatis 在预处理语句（PreparedStatement）中设置一个参数时，还是从结果集中取出一个值时， 都会用类型处理器将获取的值以合适的方式转换成 Java 类型。啥意思啊？比如我现在有个日期值，我需要这个日期值在数据库的存储形式是一串数字而不是我们日常看见的形式，但是从数据库拿出来的时候就会成为我们日常看见的形式，怎么做？这个时候就需要使用到了这个标签。

  • 首先我们需要自定义数据格式转换类继承BaseTypeHandler

    • T表示要对哪种数据格式进行转换，比如将Java类型的Date转换成数据库类型的Long型，那这个T就是Date

  • 内部覆盖4个未实现的方法。setNonNullParameter为java程序设置数据到数据库的回调方法，getNullableResult为查询时 mysql的字符串类型转换成 java的Type类型的方法

  • 在MyBatis核心配置文件中进行注册

2.6 plugins标签

MyBatis可以使用第三方的插件来对功能进行扩展，比如分页助手插件PageHelper
